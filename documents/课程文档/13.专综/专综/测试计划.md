# 介绍

本项目是一个面向娱乐的娱乐票务网站。用户可以查看最近音乐演出、舞台剧、展览等的门票销售情况，并在网上购买门票。
背景和总体目标
作为一个多元化的市场，娱乐市场发展迅速。在市场需求的带动下，网络票务平台发展迅速。该项目为需要观看音乐剧、话剧等现场表演的用户提供表演信息查看和购票服务。

# 测试套件

## 功能性test suites

### Test target

```
测试所有的用例是否正常执行
测试模块内部功能是否正常
测试模块间功能接口是否正常
测试系统是否对所有的输入输出做出了正确的相应，输出结果能否正确显示
测试系统是否存在遗漏或者多余的功能
```

### features to be tested

```
系统各个功能的执行情况，包括输入以及对应的输出是否正确
系统的功能对于用例的覆盖程度
```

### Strategies

```
用户注册
用户登录
...
```

### Tools

```
Junit
Selenium
```

### Techniques

```
White box test (Path Coverage)
Black box test (Boundary Value, Decision Table...)
```

### Pass/Fail Criteria

```
系统的所有核心功能按照预期和概述的方式运行。
系统对所有的测试输入给出了正确的输出，并正确显示输出内容。
系统不存在任何多余的功能。
```



## 安全性test suites

### Test target

```
测试账户密码的安全性
测试支付的安全性
```

### features to be tested

```
用户的账户信息是否会被泄露
用户的支付是否会出现错误
用户的支付信息是否会被泄露
系统的数据是否会被泄露
系统能否抵御一定的网络攻击
```

### Strategies

```
对软件工程文档/源代码/二进制代码进行静态分析/审核
对运行时系统进行动态监测
涉及安全的软件功能，例如：用户管理模块、权限管理、加密系统、认证系统等进行测试，主要验证上述功能是否有效
在系统本地运行特定的检测系统漏洞的程序
基于网络远程运行特定的检测目标网络和主机系统漏洞的程序
模拟攻击，使用大量半有效的数据作为应用程序的输入，以程序是否出现异常为标志，来发现应用程序中可能存在的安全漏洞
```

### Tools

```
AppScan（安全漏洞扫描工具）
Acunetix（网络漏洞扫描软件）
Fuzzing（自动化生成随机测试用例）
sqlmap（利用SQL注入漏洞，测试数据库服务器的权限）
...
```

2Techniques

```
白/灰盒测试/黑盒测试
功能验证
漏洞扫描
模拟攻击
```

### Pass/Fail Criteria

```
用户的账户信息不会被泄露
用户的支付不会出现错误
用户的支付信息不会被泄露
系统的数据不会被泄露
系统能够抵御一定的网络攻击
```

## 性能efficiency、可靠性reliability

### Test target

```
测试系统的响应性能能否达到要求
测试系统的负载能力能否达到要求（包括服务器server、数据库database...）
支付的可靠性（若支付失败，账户以及金额是否错误）
网络的可靠性（若网络错误，系统是否出现错误）
```

### features to be tested

```
系统的响应时间
系统应用延迟时间
系统的最大吞吐量
系统的最大并发用户数
系统的资源利用率
```

### Strategies

```
在真实的业务环境下运行系统，记录系统各个操作的响应时间，测试其是否处于可接受范围内。
压力测试(Stress Testing)：不断地给被测系统增加压力，直到被测系统被压垮，从而确定系统能承受的最大压力。
负载测试(Load Testing)：让被测系统在其能忍受的压力极限范围内（或临界状态下）连续运行，来测试系统的稳定性。
在给被测系统加载一定业务压力的情况下，使系统运行一段时间，以此来测试系统是否稳定。
```

### Tools

```
Jmeter（基于Java的压力测试工具）
Httperf（web 服务器的性能测试工具） 
...
```

### Techniques

```
压力测试(Stress Testing)
负载测试(Load Testing)
并发测试(Concurrency Testing)
...
```

### Pass/Fail Criteria

```
系统的响应时间在可接受范围内
系统的并发量在可接受范围内
系统的负载能力能够支撑业务的需求
```
